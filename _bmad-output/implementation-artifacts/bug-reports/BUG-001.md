# BUG-001: Cookie banner H1 captured as consultant name

**Severity:** Critical
**Status:** Fixed
**Found in:** QA Pre-testing — live scrape of `professor-stephen-mcdonnell`
**Spec reference:** Quick Spec §3.1 `consultant_name` field
**Date:** 2026-02-28

## Description

`$("h1").first()` grabs the cookie banner H1 ("Your choice regarding cookies") before the real consultant H1. The extracted name is the cookie consent text, not the consultant.

## Root Cause

Naive `$("h1").first()` selector doesn't account for cookie consent banners that inject their own H1 elements above the page content.

## Fix

Cascade selector: try `h1[itemprop="name"]` first, then `meta.swiftype[name="fullname"]`, then fall back to first H1 outside `#ccc` (cookie consent container).

## Regression Tests

- `parse.test.ts`: "skips cookie banner H1 and finds real consultant name"
