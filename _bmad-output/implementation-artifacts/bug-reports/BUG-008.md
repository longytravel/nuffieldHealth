# BUG-008: Double spaces in consultant names

**Severity:** Moderate
**Status:** Fixed
**Found in:** QA Phase 1 — Ground Truth (30-profile cohort)
**Spec reference:** Quick Spec §3.1 `consultant_name` field
**Date:** 2026-02-28

---

## Description

Multiple consultant names contain double (or triple) internal spaces, e.g. `"Dr  Laurie Windsor"`, `"Dr  Sarah  Morgan"`, `"Mr  Edward Saxby"`. The live page H1 text sometimes has extra whitespace between title and name tokens.

## Affected Profiles

| Slug | Extracted Name |
|------|---------------|
| dr-laurie-windsor | "Dr  Laurie Windsor" |
| dr-sarah-morgan | "Dr  Sarah  Morgan" |
| mr-edward-saxby | "Mr  Edward Saxby" |
| mr-jaskarn-rai | "Mr  Jaskarn Rai" |
| mr-neil-fairbairn | "Mr  Neil Fairbairn" |

## Root Cause

`extractNameAndTitle()` in `parse.ts` (line ~387) returns `trimmed` — the raw H1 text — without collapsing internal whitespace. The function detects the title prefix via regex but never normalises whitespace in the returned `name` string.

## Fix

Normalise internal whitespace before returning:

```typescript
return { name: trimmed.replace(/\s+/g, " ") || null, titlePrefix };
```

## Regression Test

Add test: H1 with double spaces → name has single spaces.
