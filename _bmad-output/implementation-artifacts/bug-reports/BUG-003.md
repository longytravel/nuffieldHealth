# BUG-003: Hospital name grabs CTA form content

**Severity:** Moderate
**Status:** Fixed
**Found in:** QA Pre-testing — live scrape of `professor-stephen-mcdonnell`
**Spec reference:** Quick Spec §3.1 `hospital_name_primary` field
**Date:** 2026-02-28

## Description

Hospital name extraction picks up CTA form content instead of the actual hospital name. `content.filter("h3")` checks the selection itself, not descendants.

## Root Cause

Wrong Cheerio API: `.filter()` tests the elements in the current set, `.find()` searches their descendants. Since `getSectionContent` returns sibling elements (not wrapped in a container), `.find()` is needed to look inside them for H3 elements.

## Fix

Changed `.filter("h3")` to `.find("h3")` and added `.first()` for safety.

- `parse.test.ts`: "extracts hospital name from H3 inside location section content"
