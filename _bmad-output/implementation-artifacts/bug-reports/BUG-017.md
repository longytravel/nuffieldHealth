# BUG-017: Duplicate title prefix on consultant detail page heading

**Severity:** Low
**Found by:** Sally (UX Designer) — visual QA
**Affected page:** `/consultants/[slug]` (all consultant detail pages)
**Spec reference:** §3.1 `consultant_title_prefix`, `consultant_name`

## Symptom

Consultant detail page heading displays **"Dr Dr Adam Lawson"** instead of **"Dr Adam Lawson"**.

The `consultant_title_prefix` ("Dr") is prepended to `consultant_name` which already contains the prefix (e.g. "Dr Adam Lawson"), resulting in a doubled prefix.

## Root Cause

In `src/app/consultants/[slug]/page.tsx` line 126-127, the template unconditionally prepends `consultant_title_prefix`:

```tsx
{c.consultant_title_prefix ? `${c.consultant_title_prefix} ` : ""}
{c.consultant_name ?? c.slug}
```

The scraper stores the full name *including* the title prefix in `consultant_name` (e.g. "Dr Adam Lawson"), and separately stores the prefix in `consultant_title_prefix` (e.g. "Dr"). The template prepends it again without checking for duplication.

## Fix

Check whether `consultant_name` already starts with the `consultant_title_prefix` before prepending it. If it does, skip the prefix to avoid duplication.

## Impact

- Visual only — affects the `<h1>` heading on all consultant detail pages where a title prefix exists
- No data integrity impact
- Affects all ~200 profiles that have a `consultant_title_prefix` value

## Verification

- Navigate to any consultant detail page with a title prefix (e.g. `/consultants/dr-adam-lawson`)
- Confirm heading reads "Dr Adam Lawson" not "Dr Dr Adam Lawson"
- Check a profile without a prefix to confirm no regression
